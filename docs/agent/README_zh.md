# CoPaw 代理系统技术文档

## 概述

本文档提供了对 CoPaw 代理系统的全面技术分析，涵盖代理循环逻辑、多代理协作模式和核心机制。CoPaw 基于 [AgentScope](https://github.com/agentscope-ai/agentscope) 构建，实现了具有可扩展钩子、内存管理和工具编排能力的单代理架构。

---

## 1. 代理架构

### 1.1 核心代理实现

主要代理类是 `CoPawAgent`，位于：

```
src/copaw/agents/react_agent.py
```

`CoPawAgent` 扩展了 AgentScope 的 `ReActAgent`，实现了 **推理 + 行动 (ReAct)** 模式。该模式将对当前状态的推理与执行工具的行动相结合，创建了一个迭代的代理循环。

#### 类层次结构

```
ReActAgent (AgentScope)
    └── CoPawAgent (CoPaw)
```

#### 关键初始化参数

| 参数 | 类型 | 默认值 | 描述 |
|-----------|------|---------|-------------|
| `env_context` | `Optional[str]` | `None` | 要前置到系统提示的可选环境上下文 |
| `enable_memory_manager` | `bool` | `True` | 是否启用内存管理器 |
| `mcp_clients` | `Optional[List[Any]]` | `None` | 用于工具集成的 MCP 客户端列表 |
| `memory_manager` | `MemoryManager \| None` | `None` | 预配置的内存管理器实例 |
| `max_iters` | `int` | `50` | 最大推理-行动迭代次数 |
| `max_input_length` | `int` | `131072` (128K tokens) | 模型上下文窗口的最大输入长度 |
| `namesake_strategy` | `NamesakeStrategy` | `"skip"` | 处理重复工具名称的策略 |

### 1.2 内置工具

CoPaw 在其工具包中注册了以下内置工具：

| 工具函数 | 描述 |
|---------------|-------------|
| `execute_shell_command` | 在主机系统上执行 shell 命令 |
| `read_file` | 从文件系统读取文件内容 |
| `write_file` | 将内容写入文件 |
| `edit_file` | 使用目标修改编辑现有文件 |
| `browser_use` | 浏览器自动化和控制 |
| `desktop_screenshot` | 截取桌面屏幕截图 |
| `send_file_to_user` | 通过渠道向用户发送文件 |
| `get_current_time` | 获取当前时间和日期 |
| `create_memory_search_tool` | 搜索代理内存（当启用内存管理器时） |

## 2. 代理循环逻辑

### 2.1 ReAct 循环模式

代理循环遵循 ReAct（推理 + 行动）模式：

```
┌─────────────────────────────────────────────────────────────┐
│                     代理循环 (max_iters)                   │
├─────────────────────────────────────────────────────────────┤
│  1. 输入：用户消息                                      │
│           ↓                                                  │
│  2. 推理：模型思考（工具选择或文本输出）               │
│           ↓                                                  │
│  3. 行动：执行所选工具或生成响应                        │
│           ↓                                                  │
│  4. 观察：获取工具结果或用户反馈                        │
│           ↓                                                  │
│  5. 内存更新：添加到对话历史                            │
│           ↓                                                  │
│  6. 检查：是否达到 max_iters 或终止状态？               │
│           ↓                                                  │
│     否 → 返回步骤 2                                         │
│     是 → 返回最终响应                                       │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 查询处理器流程

查询处理流程在 `AgentRunner.query_handler()` 中实现：

```
src/copaw/app/runner/runner.py
```

#### 查询处理管道

```
用户请求
     │
     ▼
┌──────────────────────────────┐
│ 1. 命令路径检查              │ ──── 如果查询是命令 (/restart 等)
│    _is_command(query)        │      执行命令处理器
└──────────────────────────────┘
     │
     ▼ (正常路径)
┌──────────────────────────────┐
│ 2. 构建环境上下文            │
│    build_env_context()       │
└──────────────────────────────┘
     │
     ▼
┌──────────────────────────────┐
│ 3. 获取 MCP 客户端           │ ──── 可热重载的 MCP 工具
│    _mcp_manager.get_clients()│
└──────────────────────────────┘
     │
     ▼
┌──────────────────────────────┐
│ 4. 创建 CoPawAgent           │
│    - 加载配置                │
│    - 初始化工具包            │
│    - 注册技能                │
│    - 设置内存管理器          │
└──────────────────────────────┘
     │
     ▼
┌──────────────────────────────┐
│ 5. 加载会话状态              │
│    session.load_session_state│
└──────────────────────────────┘
     │
     ▼
┌──────────────────────────────┐
│ 6. 重建系统提示              │
│    agent.rebuild_sys_prompt()│ ──── 反映最新的 AGENTS.md/SOUL.md
└──────────────────────────────┘
     │
     ▼
┌──────────────────────────────┐
│ 7. 执行代理循环              │
│    stream_printing_messages()│
│    └── agent(msgs)           │ ──── ReAct 循环
└──────────────────────────────┘
     │
     ▼
┌──────────────────────────────┐
│ 8. 保存会话状态              │
│    session.save_session_state│
└──────────────────────────────┘
     │
     ▼
   响应
```

### 2.3 会话管理

CoPaw 使用 `SafeJSONSession`（继承自 AgentScope 的 `JSONSession`）进行会话持久化：

- **会话存储**：JSON 文件位于 `{working_dir}/sessions/`
- **文件名清理**：Windows 兼容的文件名（将 `\/:*?"<>|` 替换为 `--`）
- **保存状态**：代理内存、对话历史、压缩摘要

## 3. 代理扩展和钩子

### 3.1 钩子系统

CoPaw 实现了一个钩子系统，用于在特定点扩展代理行为。注册了两个主要钩子：

#### 引导钩子

**位置**：`src/copaw/agents/hooks/bootstrap.py`

**目的**：通过检查工作目录中的 `BOOTSTRAP.md` 为首次用户提供指导。

**触发**：与代理的首次交互（推理前）

```python
# CoPawAgent 中的钩子注册
self.register_instance_hook(
    hook_type="pre_reasoning",
    hook_name="bootstrap_hook",
    hook=bootstrap_hook.__call__,
)
```

#### 内存压缩钩子

**位置**：`src/copaw/agents/hooks/memory_compaction.py`

**目的**：当内存超过阈值时自动压缩（默认：max_input_length 的 70%）。

**触发**：推理前，当内存超过 `_memory_compact_threshold`

```python
# 配置
max_input_length = 128 * 1024  # 128K tokens
MEMORY_COMPACT_RATIO = 0.7  # 70%
memory_compact_threshold = max_input_length * MEMORY_COMPACT_RATIO
```

### 3.2 命令处理器

**位置**：`src/copaw/agents/command_handler.py`

命令处理器处理用户在对话期间可以调用的系统命令：

| 命令 | 描述 |
|---------|-------------|
| `/compact` | 手动触发内存压缩 |
| `/new` | 使用摘要开始新对话 |
| `/clear` | 清除对话历史 |
| `/history` | 显示对话历史 |
| `/compact_str` | 显示压缩摘要 |
| `/await_summary` | 等待挂起的摘要任务 |

#### 命令处理流程

```
用户输入（以 / 开头）
        │
        ▼
┌─────────────────────────┐
│ is_command(query)      │ ──── 检查是否以 / 开头
└─────────────────────────┘
        │
        ▼
┌─────────────────────────┐
│ handle_command(query)   │
│   └── _process_{cmd}() │ ──── 分派到处理器
└─────────────────────────┘
        │
        ▼
┌─────────────────────────┐
│ 返回系统消息            │
└─────────────────────────┘
```

## 5. MCP（模型上下文协议）集成

### 5.1 MCP 客户端管理器

**位置**：`src/copaw/app/mcp/__init__.py`

CoPaw 支持 MCP 通过外部服务扩展工具能力：

```
┌─────────────────────────────────────────┐
│          MCPClientManager                │
├─────────────────────────────────────────┤
│  客户端类型：                           │
│  ├── HttpStatefulClient：基于 HTTP      │
│  └── StdIOStatefulClient：基于进程      │
├─────────────────────────────────────────┤
│  功能：                                 │
│  ├── 配置更改时热重载                   │
│  ├── 失败时自动重连                     │
│  └── 工具注册到工具包                   │
└─────────────────────────────────────────┘
```

### 5.2 MCP 热重载机制

```
检测到配置更改 (config.json)
        │
        ▼
┌─────────────────────────────┐
│ MCPConfigWatcher            │
│   (文件系统监视器)          │
└─────────────────────────────┘
        │
        ▼
┌─────────────────────────────┐
│ 重建 MCP 客户端             │
│   - 关闭旧连接              │
│   - 初始化新客户端          │
│   - 注册到代理              │
└─────────────────────────────┘
        │
        ▼

│ 代理获得更新的工具          │
│   (无需重启)                │
└─────────────────────────────┘
```

### 5.3 MCP 客户端恢复

代理为 MCP 客户端实现了自动恢复：

```python
# CoPawAgent 中的恢复策略
async def _recover_mcp_client(client):
    # 1. 尝试重新连接现有客户端
    if await self._reconnect_mcp_client(client):
        return client
    
    # 2. 从存储的配置重建客户端
    rebuilt_client = self._rebuild_mcp_client(client)
    if rebuilt_client and await self._reconnect_mcp_client(rebuilt_client):
        return rebuilt_client
    
    return None
```

**截至当前版本 (v0.0.4)，CoPaw 未实现多代理协作。** 系统设计为**单代理架构**，其中一个 AI 助手处理所有用户交互。

## 11. 总结

CoPaw 实现了一个复杂的单代理系统，具有：

1. **ReAct 循环**：带有可配置最大迭代次数的推理 + 行动迭代模式
2. **内存管理**：带有异步摘要的自动上下文压缩
3. **工具编排**：内置工具 + 带有热重载的 MCP 集成
4. **可扩展性**：用于预/后处理的钩子系统，技能加载
5. **会话持久化**：基于 JSON 的会话状态，支持跨平台文件名
6. **多渠道支持**：与钉钉、QQ、Discord、Telegram 等集成

**多代理协作目前尚未实现**，但已作为 AgentScope 集成路线图的一部分规划用于未来版本。
